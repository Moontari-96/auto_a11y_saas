# 📘 접근성 자동 검사 SaaS 운영 가이드 (초안)

## 1️⃣ 서버 검사 vs 브라우저 검사 역할 분리

### 🎯 목적

사이트 환경·보안 정책에 따라 **검사 방식을 명확히 구분**하여

검사 실패에 대한 혼란을 줄이고, 서비스 신뢰도를 확보한다.

---

### ✅ 서버 기반 자동 검사 (Server Scan)

### 개요

- 서버(Worker)에서 Headless Browser(Puppeteer)로 페이지를 로드하여 검사
- axe-core, Lighthouse 기반 자동 검사 수행

### 특징

- 대량 검사 가능
- CI/CD, 정기 점검에 적합
- 결과 저장 및 이력 관리 용이

### 적용 조건 (필수)

- HTTPS(SSL) 적용 사이트
- 봇/자동화 접근 차단이 없는 환경
- 고객사 소유 도메인 또는 검사 허용 환경

### 제한 사항

- SSL 미적용 사이트 ❌
- 호스팅 사업자(Gabia 등) 보안 차단 페이지 ❌
- 강력한 Bot Protection 적용 사이트 ❌

### 사용 대상

- 공공기관
- 기업 웹사이트
- 내부 관리 시스템
- 테스트/스테이징 환경

---

### ✅ 브라우저 기반 검사 (Browser Scan)

### 개요

- 사용자의 실제 브라우저 환경에서 DOM이 렌더링된 상태를 기준으로 검사
- axe-core CDN / 확장 프로그램 방식

### 특징

- 보안 정책 영향 없음
- SSL 미적용 사이트도 검사 가능
- 실제 사용자 환경 기반 검사

### 제한 사항

- 자동화/대량 검사 ❌
- 결과 이력 관리 제한
- 사용자의 수동 실행 필요

### 사용 대상

- SSL 미적용 사이트
- 서버 검사 차단 사이트
- 퍼블리셔/개발자 로컬 검수

---

### 📌 역할 분리 요약

| 구분      | 서버 검사    | 브라우저 검사   |
| --------- | ------------ | --------------- |
| 검사 위치 | 서버(Worker) | 사용자 브라우저 |
| SSL 필요  | 필수         | 불필요          |
| 대량 검사 | 가능         | 불가            |
| 차단 대응 | 제한적       | 자유            |
| 권장 대상 | B2B/공공     | 개인/퍼블리셔   |

---

## 2️⃣ “검사 불가” UX / 안내 문구 설계

### 🎯 목적

검사 실패를 **도구 문제로 오인하지 않도록**

환경적 제약임을 명확히 안내한다.

---

### ❌ 잘못된 안내 예시

```
검사 실패
에러가 발생했습니다.

```

---

### ✅ 권장 UX 문구 (서버 검사 불가 시)

### 제목

**자동 접근성 검사를 수행할 수 없습니다**

### 본문

```
이 사이트는 보안 정책 또는 호스팅 환경으로 인해
서버 기반 자동 접근성 검사가 제한됩니다.

주요 원인:
- HTTPS(SSL) 미적용
- 호스팅 사업자의 보안 차단
- 봇 접근 제한 정책

```

### 대안 안내

```
대신 다음 방법을 이용할 수 있습니다:
- 브라우저 기반 접근성 검사 실행
- 테스트/스테이징 환경에서 재시도

```

---

### 📌 내부 상태 코드 예시

```
SCAN_FAILED
FAIL_REASON:
- SSL_REQUIRED
- BLOCKED_BY_HOSTING
- BOT_PROTECTION

```

---

## 3️⃣ B2B 고객 온보딩 가이드

### 🎯 목적

B2B 고객이 **검사 환경을 사전에 준비**하도록 유도하여

도입 후 실패율을 최소화한다.

---

### ✅ 검사 가능 환경 체크리스트 (고객사 제공용)

### 필수 항목

- [ ] HTTPS(SSL) 적용 완료
- [ ] 검사 대상 도메인 고객사 소유
- [ ] 테스트 또는 운영 환경 접근 가능

### 권장 항목

- [ ] 접근성 검사 서버 IP 허용 (Whitelist)
- [ ] WAF/Bot Protection 예외 설정
- [ ] 인증이 필요한 경우 테스트 계정 제공

---

### 🔧 고객사 협조가 필요한 설정 예시

- 방화벽 IP 허용
- Cloudflare / Akamai Bot 예외
- 사내 접근성 점검용 테스트 URL 제공

---

### 📌 고객사 안내 문구 (권장)

```
본 솔루션의 서버 기반 자동 검사는
고객사 소유의 HTTPS 환경에서만 제공됩니다.

보안 정책상 서버 접근이 제한된 경우,
브라우저 기반 검사 기능을 통해 대체 검사가 가능합니다.

```

---

## 🔚 최종 요약

- 서버 검사와 브라우저 검사는 **대체 관계가 아닌 보완 관계**
- “검사 불가”는 실패가 아니라 **환경 제약**
- B2B 서비스에서는 **사전 가이드가 곧 품질**

---

### 🧠 한 줄 핵심 메시지

> 자동 접근성 검사는 기술 문제가 아니라
>
> **환경·정책·책임 범위를 명확히 하는 서비스 설계의 문제**다.
